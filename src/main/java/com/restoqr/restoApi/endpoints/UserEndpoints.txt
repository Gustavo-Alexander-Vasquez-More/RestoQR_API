# Endpoints de Usuario - RestoQR API

GET    /users/
    \- Descripción: Lista todos los usuarios registrados.
    \- Respuesta: Lista de UserDTO (sin contraseñas).

GET    /users/find/{username}
    \- Descripción: Busca un usuario por su nombre de usuario.
    \- Respuesta: UserDTO si existe, mensaje de error si no.

GET    /users/count
    \- Descripción: Devuelve el número total de usuarios registrados.
    \- Respuesta: Objeto con el campo "count".

GET    /users/exists/{id}
    \- Descripción: Verifica si existe un usuario por su id.
    \- Respuesta: Objeto con el campo "exists" (true/false).

POST   /users/create/{username}/{password}/{role}
    \- Descripción: Crea un nuevo usuario con nombre, contraseña y rol.
    \- Validación: No permite nombres de usuario duplicados.
    \- Respuesta: UserDTO del usuario creado o mensaje de error.

POST   /users/login?username={username}&password={password}
    \- Descripción: Inicia sesión y devuelve un token JWT si las credenciales son correctas.
    \- Respuesta: Objeto con el campo "token" o mensaje de error.

DELETE /users/delete/{id}
    \- Descripción: Elimina un usuario por su id.
    \- Respuesta: Mensaje de éxito o error.

PUT    /users/update/{id}/{username}/{password}/{role}
    \- Descripción: Actualiza los datos de un usuario existente por id.
    \- Respuesta: Mensaje de éxito o error.
