# Endpoints de Productos - RestoQR API

POST   /products/create
    - Descripción: Crea un nuevo producto. Si la categoría es "drink" crea una bebida, si es "dish" crea un plato.
    - Requiere:
        Para bebida (drink):
            {
                "nameProduct": String,
                "description": String,
                "price": Double,
                "category": "drink",
                "stock": Integer,
                "isAlcoholic": Boolean,
                "available": Boolean
            }
        Para plato (dish):
            {
                "nameProduct": String,
                "description": String,
                "price": Double,
                "category": "dish",
                "subcategory": Integer, // 1: entrada, 2: plato principal, 3: postre
                "isVegetarian": Boolean,
                "available": Boolean
            }
    - Respuesta: Producto creado o mensaje de error.

GET    /products/search/{nameProduct}
    - Descripción: Busca productos cuyo nombre contenga el texto ingresado (búsqueda flexible, insensible a mayúsculas/minúsculas).
    - Respuesta: Lista de productos coincidentes o mensaje de error.

GET    /products/all
    - Descripción: Lista todos los productos registrados.
    - Respuesta: Lista de productos.

GET    /products/count
    - Descripción: Devuelve el número total de productos registrados.
    - Respuesta: { "count": número }

PUT    /products/update/{id}
    - Descripción: Actualiza los datos de un producto por su id. Actualiza campos específicos si es bebida o plato.
    - Requiere: JSON con los datos actualizados.
    - Respuesta: Mensaje de éxito o error.

DELETE /products/delete/{id}
    - Descripción: Elimina un producto por su id.
    - Respuesta: Mensaje de éxito o error.
